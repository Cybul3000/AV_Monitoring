# Implementation Plan: Dante Network Audio Device Monitoring and Subscription Management

**Branch**: `003-dante-network-audio` | **Date**: 2026-03-24 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/003-dante-network-audio/spec.md`

## Summary

Implement a Dante network audio device module for the AV monitoring app. The module discovers Dante devices on the local network using mDNS (four `_netaudio-*._udp.local` service types), queries device state and channel lists via the Dante ARC protocol (UDP port 4440, binary big-endian packets), monitors device liveness via multicast heartbeats (224.0.0.233:8708), and surfaces routing subscription health through the standard LED status system. Write and remove subscriptions, update device settings (sample rate, encoding, latency), rename devices and channels, and control AVIO gain levels via dedicated IPC channels.

The Dante ARC/Settings protocol is proprietary and reverse-engineered (from `netaudio-lib`). No npm package implements it — the transport is ported from Python to TypeScript using Node.js built-ins (`dgram`, `Buffer`, `os`) plus `multicast-dns` for Zeroconf discovery.

## Technical Context

**Language/Version**: TypeScript 5.x (Electron 30 main process, Node.js 20 LTS)
**Primary Dependencies**: `multicast-dns` (new, npm) + `dgram` / `Buffer` / `os` (built-in Node.js)
**Storage**: SQLite (`better-sqlite3`) — migration `005_dante.sql` adds `dante_devices`, `dante_channels`, `dante_subscriptions`
**Testing**: Vitest (unit + integration), Playwright (E2E — Agent H scope)
**Target Platform**: macOS 12+ and Windows 10+ (cross-platform, no native bindings)
**Project Type**: Electron desktop app module (same pattern as existing device modules)
**Performance Goals**: Discover all devices on a ≤50-device network within 10 s (SC-001); all read operations return within 5 s (SC-004)
**Constraints**: UDP multicast must work on both macOS and Windows; no Python runtime dependency; no auth on Dante wire protocol (ARC port 4440)
**Scale/Scope**: ≤50 Dante devices per network segment; up to 64 channels per device (paginated); one subscription per RX channel
